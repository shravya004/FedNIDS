# Federated IDS — FLNET2023 — Member 1 (Dataset & Hybrid Model)

Base paper: *PoisonShield-FL-NIDS* (FL + trust-aware aggregation + poisoning defense)
Project: Robust Explainable Federated Intrusion Detection Against Model Poisoning
using Adaptive Trust-Aware Aggregation and Hybrid Deep Learning.

This repo contains **Member 1's** scope:

1. **FLNET2023 dataset processing** — cleaning, feature selection, per-client
   train/val splitting that mirrors the dataset's real (non-IID) D1–D10
   router topology, and an aligned global test set.
2. **`HybridGRUTransformer`** (`src/models/hybrid_model.py`) — the shared
   detection backbone, plus `GRUClassifier` / `TransformerClassifier`
   baselines for the comparison table.

Everything here is designed to be imported directly by **Member 2**
(trust scoring) and **Member 3** (Flower federated setup) without any
changes on their end.

## 3. Running it — two paths

### Path A: instant test with synthetic data (no download needed)

```bash
python scripts/generate_synthetic_data.py   # fake FLNET2023-shaped data, ~5 sec
python scripts/run_preprocessing.py         # full clean -> split pipeline
pytest -q                                   # model unit tests
python scripts/sanity_check_train.py --client_id 1 --epochs 5
```

This proves the whole chain works before you touch the real (multi-GB)
dataset.

### Path B: real FLNET2023 data

1. Download the dataset from the link in the dataset authors' README:
   https://github.com/nsol-nmsu/FML-Network
2. Place it so the structure looks like:
   ```
   data/raw/FLNET2023/
       Normal/CSV/Dataset-<id>.csv
       DDoS/CSV/Dataset-<id>-<TrafficType>.csv
       DoS/CSV/Dataset-<id>-<TrafficType>.csv
       Infiltration/CSV/Dataset-<id>.csv
       Web/CSV/Dataset-<id>-<TrafficType>.csv
       TEST/CSV/*.csv
   ```
3. Run the same two commands:
   ```bash
   python scripts/run_preprocessing.py
   python scripts/sanity_check_train.py --client_id 1 --epochs 10
   ```

The dataset is several GB (DDoS captures alone run tens of GB in PCAP —
the CSV folders are much smaller and are all you need). Only the `CSV/`
subfolders are read; `PCAP/` is ignored.

## 4. What gets produced

| File | Description |
|---|---|
| `data/processed/flnet2023_clean.csv` | full cleaned, scaled, labeled dataset |
| `data/processed/clients/client_<id>/train.csv` | per-client training shard |
| `data/processed/clients/client_<id>/val.csv` | per-client validation shard |
| `data/processed/clients/clients_manifest.json` | row counts + class balance per client |
| `data/processed/test/global_test.csv` | held-out global test set, same scaler/features as training |
| `artifacts/scaler.joblib` | fitted `StandardScaler` |
| `artifacts/metadata.joblib` | class names + final feature column list |

Client partitioning maps 1:1 onto the dataset's real router nodes
(D1–D10), so the splits are genuinely non-IID — useful for both FL
realism and Member 2's poisoning/trust experiments.

## 5. For Member 3 (Federated Learning lead)

```python
from src.datasets.flnet_dataset import load_client_loaders
from src.models.hybrid_model import HybridGRUTransformer

train_loader, val_loader, seq_len, input_dim = load_client_loaders(client_id=3, batch_size=64)

model = HybridGRUTransformer(input_dim=input_dim, seq_len=seq_len, num_classes=5)

# Flower-ready weight helpers:
weights = model.get_weights()      # list[np.ndarray] for get_parameters()
model.set_weights(weights)         # for set_parameters()
```

`load_global_test_loader()` in the same module gives you the aligned
held-out test set for global-model evaluation each round.

## 6. For Member 2 (Adaptive Trust Scoring lead)

`model.forward_features(x)` returns the pooled embedding **before** the
classifier head — useful if you want per-client embedding comparisons,
or to feed an explainability layer (SHAP/LIME) downstream. `get_weights()`
gives you the same flattened parameter view Flower uses, if you want to
compute gradient-similarity trust signals directly on parameter deltas.

## 7. Model architecture (`HybridGRUTransformer`)

```
Input (batch, seq_len, input_dim)
        ↓
   Bidirectional GRU  (2 layers)
        ↓
   Linear projection -> d_model
        ↓
   Positional Encoding
        ↓
   Transformer Encoder (2 layers, multi-head attention)
        ↓
   Mean / Last / CLS pooling
        ↓
   LayerNorm -> Dense -> GELU -> Dropout -> Dense
        ↓
   Class logits
```

All dimensions are constructor arguments — see the docstring in
`src/models/hybrid_model.py` for the full parameter list and defaults.

## Adaptive Trust Framework (Member 2)

The Adaptive Trust Framework enhances the security of the federated intrusion detection system by identifying unreliable or malicious clients before global model aggregation.

### Components

#### Adaptive Trust Scoring

Each participating client receives a trust score based on multiple factors:

* Embedding similarity between the client model and the global model
* Local validation accuracy
* Historical reputation
* Anomaly score

#### Reputation Memory

A reputation manager maintains historical trust information for every federated client across communication rounds.

Features include:

* Current reputation
* Trust history
* Average reputation
* Communication round tracking
* Consecutive low-trust counter

#### Automatic Blacklisting

Clients with consecutive low trust scores are automatically blacklisted and excluded from future aggregation rounds.

Stored information includes:

* Client ID
* Blacklisting reason
* Communication round
* Trust score

#### Trust-Aware Aggregation

Instead of conventional FedAvg, client model updates are weighted according to their trust scores. Blacklisted clients are excluded from aggregation.
